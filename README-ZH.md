# OriginTab

## 安装

请访问 <https://products.g1en.site/zh/origintab> 安装适用于不同浏览器的 OriginTab。

## 简介

如果你想要一种简单直接的方式来保存浏览器标签页，并在需要时恢复它们，那么 OriginTab 就是为你设计的。

这是一个 Vibe coding 产品，旨在复制最初的 [OneTab](https://www.one-tab.com/)，它移除了当前版本的 OneTab 的所有复杂功能。OriginTab 只专注于保存和恢复，没有任何其他复杂的概念。

虽然你可以在 OriginTab 中创建多个分组，但这不是必需的，因为所有分组都会根据保存时间创建子分组。OriginTab 安装后，会创建一个你可以直接使用的默认分组。

OriginTab 的设置会在浏览器间同步，但标签页只保存在本地。你可以随时导入和导出它们。

## 为什么创建它，而不是使用 OneTab

我使用 OneTab 已经很久了，我用它来临时保存这样的标签页：

- 有些我现在没时间查看，但几天后会看的。
- 目前打开的标签页太多了，但其中一些我后面还会用到，所以我会暂时把它们都放到 OneTab 里来提高浏览器速度。

就这样。我没有任何复杂的需求。我觉得之前版本的 OneTab 完美满足了我的需求；它的界面简洁，功能直接。但最近的重新设计完全颠覆了它。

现在的 OneTab 充满了莫名其妙的特性：归档？回收站？子文件夹？我觉得很多人不会用这些；至少我从来没用过，也从来没创建过任何文件夹。我一直都用默认文件夹。

我想要的是一个极其简单的工具，用于保存临时标签页，但 OneTab 已经不再是那样了。

这就是我创建 OriginTab 的原因。

## 一些默认设定

- OriginTab 允许重复项（大多数时候你不会在乎是否有重复）。
- OriginTab 不会保存固定的标签页（它假定固定标签页是永驻的）。
- OriginTab 默认会将标签页保存到最近创建的分组中。
- 当恢复标签页组的时候，OriginTab 会自动冻结它们，以减少内存占用。

## 快捷键

- 打开 OriginTab：默认为 `Ctrl + R`
- 保存所有标签页：需要手动设置
- 保存当前标签页：需要手动设置
